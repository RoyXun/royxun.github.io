X-UA-Compatible IE=edge,chrome=1

发表于 2016-04-13   |   分类于 浏览器兼容   |  

之前做的几个项目对浏览器要求不是很高，IE只要样式不要错乱得离谱就行，低版本干脆直接忽略。遇到需要兼容高版本IE时，就在 head里添加meta信息:

加上这句就能让IE表现得像个正常浏览器，所以一直没深究过X-UA-Compatible的用法。直到最近接手新项目，发现这句设置居然莫名其妙地失效,才开始各种查阅它的相关用法。

项目要求兼容IE9，便在条件注释中判断浏览器版本，如果低于IE9就重定向到浏览器不兼容提示页面。meta信息设置了，代码好像也写得没错，但是当用IE9去访问测试环境页面时，发现页面居然默认开启兼容模式而被重定向了。对于这种最不拿手的兼容性问题，当然是谷歌了。果然不负众望，在MSDN中发现这么一句：

The X-UA-Compatible header isn’t case sensitive; however, it must appear in the header of the webpage (the HEAD section) before all other elements except for the title element and other meta elements.

即

X-UA-compatible 标头不区分大小写；不过，它必须显示在网页中除 title 元素和其他 meta 元素以外的所有其他元素之前的标头中。

检查项目代码，发现负责埋点的同事把<script>放到了head开头、<meta>前面，于是把埋点信息挪到<body>尾部，整个世界清静了……

再来大致说下IE=edge,chrome=1到底是何含义。

Google开发了一个Google Chrome Frame(GCF),可以让用户的IE浏览器外观不变，但在浏览网页时实际上使用的是Chrome的内核。如果用户安装了GCF，则使用GCF渲染页面chrome=1;否则，用最高版本的IE内核渲染IE=edge。不过GCF在2014年已经退休了，所以理论上来说设置浏览器兼容只需要IE=edge一条就够了。至于X-UA-Compatible的取值，可以参照X-UA-Compatibility Meta Tag and HTTP Response Header,在此不再赘述。

PS: 改完兼容性bug之后，惊奇地发现99css在六年前踩过相同的坑（IE8 X-UA-Compatible 失效？），要是能早点搜到这篇博客，就不用那么辛苦地各种排查各种查阅了。年年岁岁坑相似，就记录一下这个坑，作为个人博客的处女篇，希望能为后来者提供点帮助吧。
